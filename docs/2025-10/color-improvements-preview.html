7. Implementation Checklist ✓

  • All interactive text/background pairs meet WCAG contrast in both light & dark
  • Accent colors internally harmonious (reduced chroma, coherent hue relationships)
  • Number of "strong" hues limited to 5 (neutral, brand green, teal, purple, amber)
  • Intermediate states exist for hover, active, disabled (brand-650, brand-750, *-light variants)
  • Chart colors sufficiently separated and accessible (neutral baseline, brand emphasis)
  • Dark mode neutrals and accents visually balanced (desaturated accents, no pure black)
  • Background tints added for layering without saturation (chart-*-bg variants)

Summary of Changes

🆕 New Variables Added (8 total)

  • --brand-650: oklch(60% 0.11 125) — Hover state for primary buttons
  • --brand-750: oklch(45% 0.095 125) — Active/pressed state
  • --accent-purple-light: oklch(85% 0.10 295) — Soft purple backgrounds
  • --accent-teal-light: oklch(90% 0.06 180) — Soft teal backgrounds
  • --chart-disabled: oklch(85% 0.005 285) — Muted/inactive chart data
  • --chart-1-bg through --chart-5-bg: Background tints for layering

🔧 Modified Variables (4 total)

  • --accent-purple: Chroma reduced 0.25 → 0.20 (less saturated)
  • --accent-teal: Chroma reduced 0.15 → 0.12 (better harmony)
  • --neutral-400: Lightness adjusted 63.9% → 66.0% (slightly warmed mid-tone)
  • Dark mode accents: All chroma values reduced by ~0.02-0.05 for less glare
